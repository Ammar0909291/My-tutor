# The Lesson Planning Engine

The decisions above the turn: what this session contains, what the next
one will, and how the plan bends when the learner diverges from it. The
plan serves the learner; the learner never serves the plan.

## 1. The session shape (the default template)

```
OPEN     due-review retrieval first (memory engine), incl. yesterday's
         C2 check — before ANY new content. 2–5 minutes.
CORE     ONE of: a new concept pass · a consolidation pass · a
         misconception repair pass · a re-gate. One. The session has
         one job; second jobs are how sessions fail.
STRETCH  optional, offered-not-assigned, only from confident+curious
         with the core complete.
CLOSE    scripted, protected, never sacrificed to content.
```

Time boxes scale with the learner (child beginner 5–10 min → established
adult 30–40), but the SHAPE is invariant, and overruns are always cut
from CORE repetitions and STRETCH — never from OPEN's retrieval or the
CLOSE.

## 2. Continue / repeat / slow down (the default triad)

- **CONTINUE** (advance to the next machine step / next concept) on the
  fluency gate only: fast + correct + confident, three in a row for
  step-level advances (Delivery 1); the full evidence set for
  concept-level gates (`../assessment/05 §3`). Correctness alone never
  advances anything (Universal Principle 3/22).
- **REPEAT** (another pass at the same step, fresh isomorphs) on
  slow-correct or hedged-correct — consolidation, not punishment, and
  never announced as repetition ("same skill, new one" — the surface
  changes even when the step doesn't; identical-item repetition is
  banned, `../assessment/09 §4`).
- **SLOW DOWN** (halve step size, add a sub-step, drop transfer
  distance) on: two consecutive failures at one step, any load signal,
  or a failure plus frustration (D1 rule). Reduce ONE dimension —
  smaller numbers or fewer steps or recognition-instead-of-recall,
  never "an easier topic" (audible demotion).

## 3. Skip / accelerate (the compaction protocol)

Triggers: BORED with perfect performance; false-beginner signatures
(`../first-lesson/01 §4`); returning learners cascading; gifted-persona
entry (Delivery 2 §9).

```
COMPACTION
1. Offer the test-out honestly: "I think you know some of this —
   want to prove it and skip ahead?" (Offered — never imposed;
   some learners want the run-up.)
2. Pretest with GATE-grade items (production, new-surface, one
   mixed), not recognition — compaction on recognition evidence
   builds the tower on sand.
3. Skip what passes; the skipped concepts enter the review
   schedule as if gated (they still decay!), at long intervals.
4. First failure ends the skip run — land there, that's the
   frontier (binary search logic, ../assessment/02).
5. Never skip past a cut-node without testing it explicitly
   (prerequisite fan-out makes cut-node gaps catastrophic).
```

## 4. Review injection and prerequisite revisits

- **Reviews are due-date-driven** (memory engine, Delivery 2 §8): due
  items open the session, mixed across subjects; failure demotes the
  interval one step, fast success expands it. The plan never skips due
  reviews to "make progress" — skipped reviews are borrowed progress
  repaid with interest.
- **Prerequisite revisits are trigger-driven**, not scheduled: the
  three-representation rule, systematic-error patterns, or an
  assessment autopsy naming the floor (`../assessment/09 §3`). The
  revisit interrupts the plan by design ("let's sharpen the tool
  first"), teaches the floor to genuine fluency (not a drive-by
  mention), and returns via isomorph in a LATER session.
- **Misconception repairs get their own CORE slot** — never appended to
  the end of another lesson (the collide-and-contrast sequence needs a
  fresh learner, Delivery 2 §4; scheduling it at minute 25 wastes it).

## 5. Introducing challenge

- Trigger: gate passed + drive-band state (confident/curious) —
  the excitement moment is the stretch moment (Delivery 1).
- Form: ONE item beyond the current gate — a far-transfer surface, the
  weird case, the "construct your own example" inversion.
- Framing: adventure, not evaluation ("want to see the mean version?"),
  and a miss costs nothing visible ("that one's next week's — you just
  previewed it").
- Persona tuning: competitive and gifted learners get the stretch
  earlier and harder (challenge IS their engagement channel); fearful
  learners get it later, privately, after the repair path has reached
  CONFIDENT (02 §3).

## 6. Ending the lesson

Hard triggers (end now, via a win and the close):
- affect budget spent (2 visible failures; 1 in lesson one);
- recovery rung 1 (05 §5);
- fatigue trend confirmed;
- the time box.

Soft trigger — the one tutors most often miss: **the summit**. When the
session's one job lands early — the solo success, the gate passed, the
misconception collapsed — strongly prefer ending there, even with time
remaining. "Bank the win" (`../first-lesson/04 §1`) generalizes: minutes
after a summit add risk, not value, and a learner who leaves at the top
returns hungry. The plan's unused material is tomorrow's OPEN.

## 7. The multi-session arc

- **A concept is STUCK** after 2 failed gates → teaching-path defect
  protocol (05 §4): family change, spacing shrunk, feedback artifact
  authored.
- **Block early, interleave forever**: a concept's first sessions may be
  blocked (stability needs density); from first gate onward its items
  live only in mixed sets (Delivery 2 §8).
- **The arc has a visible horizon**: learners are told where they are in
  plain terms ("two more like this and fractions can't touch you") —
  progress made visible is motivation that costs nothing; progress
  hidden in tutor bookkeeping motivates no one.
- **Plans decay too**: a plan older than the learner's last three
  sessions of evidence is stale — re-derived from the current ledger,
  not patched. The learner model is the source of truth; the plan is a
  cached view of it.
