# The Escalation Engine

What to do when the thing you tried fails — and when IT fails, and when
that fails. Escalation is a ladder per failure type, with three universal
laws before any ladder:

1. **Each rung changes exactly ONE dimension** (representation, size,
   modality, or level — never two at once; a double change that works
   teaches you nothing about why).
2. **Never skip rungs upward, always allowed downward.** Escalating
   straight from one failed explanation to "end session" wastes a
   learner who needed only a demonstration.
3. **Count everything.** Every escalation is a feedback-loop signal
   (`../first-lesson/08` generalizes to all lessons): rung 3+ on any
   ladder for any concept is an authoring flag against that concept's
   entry.

## 1. When an EXPLANATION fails

("I don't understand", confusion signals, or a wrong echo of the idea.)

```
RUNG 1  Different FRAME, same channel — verbal again but story-words
        instead of mechanism-words (or vice versa). ONE retry. Never
        the same sentence louder (Universal Principle 2).
RUNG 2  Change CHANNEL: demonstrate. Words were pointing at an
        experience the learner doesn't have — give the experience
        (../first-lesson/04 §2, true at every level).
RUNG 3  Concrete enactment: the learner's hands/body/objects — below
        demonstration in abstraction.
RUNG 4  STOP explaining this concept. Three representations have
        failed → the floor is the problem (D1 three-representation
        rule): run prerequisite micro-diagnosis
        (../assessment/02 §9) and teach what it finds.
```
There is no rung 5. A fourth explanation of the same concept to the same
learner in the same session does not exist as a legal move.

## 2. When an ANALOGY fails

(Needs defending twice, or produced a wrong inference.)

```
RUNG 1  One patch is legal ("in this analogy, ignore the...").
RUNG 2  Two patches = the mapping is broken for this learner: SWAP
        to an analogy with a DIFFERENT SOURCE DOMAIN (money→water is
        a real swap; two water variants are cousins, not a swap).
        Consult the concept entry's ranked library first.
RUNG 3  Drop analogy as a family for this concept-learner pair: go
        direct — demonstration or worked mechanism. Some pairs don't
        take analogies; record the affinity (04, filter 5).
ALWAYS  Check the entry's ANTI-ANALOGY list: if the failed analogy is
        documented as a backfire, the failure was authoring debt —
        repair the learner's imported inference NOW (it's a fresh
        misconception, birth type 6) and flag the source that used it.
```

## 3. When a DEMONSTRATION fails

(The learner watched and nothing landed, or predicted wrong and shrugged.)

```
RUNG 1  Re-run SLOWED, with the critical instant exaggerated and a
        committed prediction attached ("watch RIGHT here — call it
        first"). A demo without a stake is television.
RUNG 2  DECOMPOSE: demo one component alone (the friction-dial's
        single surface; one fold of the paper).
RUNG 3  Transfer the apparatus: the LEARNER runs it (watching can
        fail where doing can't — Universal Principle 19).
RUNG 4  Prerequisite check — a demonstration failing three forms
        usually means the learner lacks the perceptual prerequisite
        to see what's being shown (they can't see the "change of
        motion" because velocity-as-a-thing isn't there yet).
```

## 4. When an ASSESSMENT fails

Fully owned by `../assessment/09` (six-cause taxonomy, 60-second autopsy,
re-assessment rules, circuit breaker). This engine adds only the
escalation position above it:

```
1st gate failure on a concept   → assessment/09 flowchart; remediate
                                  the found cause; re-gate delayed.
2nd gate failure, same concept  → TEACHING-PATH DEFECT declared: the
                                  method is wrong for this learner,
                                  not the learner for the method
                                  (Universal Principle 23). Change
                                  representation FAMILY for the
                                  reteach, shrink spacing, and author
                                  the feedback-loop artifact this week.
3rd                             → the concept's prerequisite CHAIN is
                                  suspect, not the concept: re-verify
                                  the whole local floor (binary search
                                  down, assessment/02), and check the
                                  KG edge list — a persistent
                                  unexplained failure is how hidden
                                  prerequisites announce themselves
                                  (curriculum feedback channel).
```

## 5. When RECOVERY fails

(The Recovery Engine ran and the learner is still flooded or shut down.)

```
RUNG 1  Stop repairing, start closing: end-session protocol — the
        tutor performs something enjoyable with zero demand, banks
        the tiniest available win, runs the close script early.
        A shortened session that ends warm is a SUCCESS of this rung.
RUNG 2  Next-session rebuild: plan halved, success density raised
        (~95%), opening retrieval chosen to be unmissable. The
        recovery didn't fail because scripts were weak — the affect
        debt was bigger than one session; pay it across several.
RUNG 3  Recovery failing at the START of two consecutive sessions →
        persona reclassification (the working model of this learner
        is wrong — likely fearful or burned-out mislabeled as
        unmotivated; Delivery 2 §9 diagnosis-before-treatment), and
        the multi-session arc is rebuilt from that diagnosis.
NEVER   Escalate recovery by intensifying it in-session — more
        cheerleading, more pep, more "let's try once more" is
        pressure wearing a smile. Recovery escalates by SHRINKING
        the session, never by working the learner harder.
```

## 6. The six standing moves, with their triggers

The moves every ladder draws from — defined once:

| Move | Trigger | Rule |
|---|---|---|
| **REPEAT** | the attempt was interrupted, mis-heard, or the learner asks | legal exactly once, and only for delivery problems — never as a treatment for non-understanding |
| **CHANGE REPRESENTATION** | genuine non-understanding, first and second occurrence | new FRAME first, then new CHANNEL; same-frame paraphrase is not a change |
| **STEP BACK** | struggle at the current machine step; solo fails; confusion in GUIDED | one step / one sub-step, silently, no commentary ever (01 §3) |
| **STEP FORWARD** | fluency signal (fast+correct+confident), boredom, spontaneous production | one step; skipping several requires the compaction protocol (07 §3), not enthusiasm |
| **CHANGE MODALITY** | comprehension stalls with motivation intact; channel-specific failure (can say it, can't write it) | to the concept's next legal channel (04 filters 3–4); also run the modality cross-check before concluding anything from a single-channel failure (../assessment/06 §4) |
| **CHANGE PREREQUISITE (step down a level)** | three-representation rule; systematic errors; assessment autopsy names the floor | via micro-diagnosis (../assessment/02 §9), teach what it finds, return LATER via isomorph — never same-session return to the failed concept |
| **END SESSION** | affect budget spent (2 visible failures; 1 in lesson one); recovery rung 1; fatigue trend; time box | ALWAYS through a banked win and the close script — ending is a move done well, not a move that failed |
