# The Misconception Birth Taxonomy

Six mechanisms by which a wrong mental model forms — cited by number
throughout this tree ("birth type 1" through "birth type 6") since the
first concept entries were written, never before collected in one place.
`../student-state/03-misconception-ledger.md §3` already reads a
regression-risk prior FROM this taxonomy for every ledger entry; this
file is the taxonomy that table depends on.

**Why birth type matters more than "the learner is wrong":** a
misconception is not an absence of knowledge — it is a working, self-
defending model that produces systematic wrong answers. The correct
repair depends entirely on WHY the model formed, not just on what it
gets wrong. A repair aimed at the wrong mechanism (verbal argument
against a perceptually-grounded belief; a third representation against a
notation artifact) fails, and a failed repair is worse than no repair —
it teaches the learner that the correct answer doesn't actually hold up
to scrutiny.

## 1. Type 1 — Overgeneralization

**Mechanism**: a rule that is genuinely correct in a prior domain gets
extended into a new domain where it silently breaks. The learner is not
being careless — they are applying a real, previously-validated rule to
a case that looks similar on the surface but violates the rule's actual
boundary condition, which was never explicitly marked.

**Diagnostic signature**: the wrong answer is rule-governed, not random
— the learner can usually state the rule, and the rule is demonstrably
TRUE somewhere else in their existing knowledge. Ask about the old
domain directly and the learner will typically get it right; the error
only appears at the boundary crossing.

**Cross-subject examples**:
- Mathematics: "a bigger denominator means a bigger fraction" (whole-
  number ordering — bigger numbers are bigger — extended past the point
  where it holds).
- English: regular past-tense formation ("add -ed") extended to
  irregular verbs — "goed" instead of "went," "runned" instead of "ran."
  This is not a spelling slip; it is the correct English rule for the
  majority of verbs, applied exactly where it stops working.

**Repair implication**: name the old rule's home domain explicitly and
out loud ("that's exactly right for whole numbers — watch what happens
here"), then locate the boundary precisely with the learner ("where does
that rule's country end?") rather than simply declaring the new domain's
rule. The old rule is not wrong; it is scoped, and the scoping is the
lesson.

## 2. Type 2 — Perceptual Intuition

**Mechanism**: built from thousands of unlabeled, never-explicitly-
taught sensory observations across a lifetime, all of which happened to
share a hidden confound. Unlike Type 1, there is no statable "rule" the
learner learned somewhere — the model is diffuse, built from raw
experience, and was never taught by anyone. This is the deepest and most
resistant birth type.

**Diagnostic signature**: fires even under careful, unhurried reasoning
and even in learners who can correctly STATE the right answer — the
correct answer is effortful and consciously constructed; the wrong
answer "just feels obviously true." Regrowth under time pressure or
fatigue is the best-documented pattern of any birth type — this is a
model competing for the same cognitive real estate as the correct one,
not a model that gets replaced.

**Cross-subject examples**:
- Physics: "an object needs a continuous force to keep moving" (impetus
  theory) — built from a lifetime of watching thrown, pushed, or rolled
  objects slow down, because friction is present in essentially every
  observation a person makes before physics class.
- Chemistry: "solids are always denser than liquids" — built from
  everyday experience of solid objects (rocks, wood, metal) that mostly
  do sink in water — contradicted by ice, which is a solid that floats
  on its own liquid.

**Repair implication**: verbal argument does not work against this type
— the model was never built from words and will not be dismantled by
them. The collision must be physical or directly observed (drop the ice
in water; roll a ball on a genuinely low-friction surface), and even
after a successful collision, expect this type to resurface under speed
or load indefinitely — per `../student-state/03 §3`'s prior, this type
never fully dies, and verification schedules should treat it as
permanent rotation, not a one-time repair.

## 3. Type 3 — Language Contamination

**Mechanism**: an ordinary word carries one meaning in everyday
conversation and a different, more precise meaning in the discipline.
The learner's everyday sense of the word intrudes on the technical
context specifically because the SAME WORD is doing both jobs.

**Diagnostic signature**: the error correlates with the specific word
choice, not with the underlying difficulty of the content — rephrase the
question to avoid the contaminating word entirely, and the error
frequently disappears. This is the cleanest diagnostic test in the whole
taxonomy: a misconception that survives a same-difficulty rewording with
different vocabulary is NOT type 3.

**Cross-subject examples**:
- Mathematics: "reduce a fraction" (6/8 → 3/4) sounds like the fraction
  is getting SMALLER in value, because "reduce" means "shrink" in
  ordinary usage — the value is unchanged; only the representation
  simplifies.
- Biology: "it's just a theory" — everyday "theory" means an unproven
  guess; scientific "theory" (as in the theory of evolution) means a
  well-substantiated explanatory framework with extensive supporting
  evidence. The contamination runs in the politically consequential
  direction here, which makes it unusually persistent.

**Repair implication**: name the collision explicitly as a word problem,
not a content problem ("this is one of those words that means something
different in maths than it does at home — let's separate the two
meanings"), and re-check specifically by presenting the same content with
the contaminating word removed or replaced.

## 4. Type 4 — Notation-Induced

**Mechanism**: the way something is WRITTEN has a visual or positional
logic that, read naively, suggests an incorrect structure. The
misconception is a byproduct of the symbol system itself, not of the
underlying idea — a learner encountering the same content through speech
or physical manipulation would not necessarily form the same wrong model.

**Diagnostic signature**: the error is characteristic of the written
symbolic form specifically. The strongest diagnostic test: present the
identical content without the symbolic notation (orally, or with
physical objects) — a genuinely notation-induced error frequently
vanishes.

**Cross-subject examples**:
- Mathematics: the fraction bar (1/2 + 1/3) visually reads as "two
  numbers stacked, add the tops and add the bottoms" rather than as one
  number.
- English: apostrophe placement in "its" vs. "it's" — the general
  written rule "add 's to show possession" is itself taught via
  notation, and "it's" visually LOOKS like it should follow that same
  pattern, producing the reverse error (writing "it's" for the
  possessive) at least as often as the standard error.

**Repair implication**: change the representation entirely — move off
the page. A notation-induced error is rarely solved by a clearer verbal
explanation of the SAME symbols; it is solved by temporarily removing
the symbols and rebuilding the correct structure in a different medium,
then reintroducing the notation once the underlying structure is secure.

## 5. Type 5 — Instruction-Induced

**Mechanism**: teaching necessarily begins with a narrow, correct-but-
incomplete case for accessibility (whole numbers before negatives, one
whole pizza before improper fractions, one indivisible particle before
subatomic structure). If the boundary of that narrow case is never named
at the time it is taught, the learner reasonably concludes the narrow
case IS the general rule — because nothing ever told them otherwise.

**Diagnostic signature**: unlike Type 2 (which varies by individual
lived experience), Type 5 is a signature of the CURRICULUM, not the
individual — it appears near-universally across an entire cohort taught
by the same method, and largely disappears when a different curriculum
names the shelf-life explicitly at the point of teaching (Universal
Principle 6, `../foundations/04-universal-teaching-principles.md`: "a
representation's scope of validity must be named at installation, not
discovered by the learner alone").

**Cross-subject examples**:
- Mathematics: "a fraction can't be bigger than 1" — built entirely from
  early examples that were always one whole pizza, cut into equal
  slices, with no case ever shown where the numerator exceeds the
  denominator.
- Chemistry: "atoms are the smallest indivisible unit of matter" —
  taught as settled fact in early chemistry, without the qualifier "at
  the scale that matters for chemical bonding," so subatomic particles
  arrive later as an apparent contradiction rather than a natural
  refinement.

**Repair implication**: this is the one birth type that is entirely
preventable at the point of original teaching — name the shelf-life the
first time the simplified model is introduced (Principle 6), and Type 5
never forms. Where it has already formed, treat it exactly as a
promotion, not a correction: "this picture needs to grow now — that's
not you being wrong, that's the next stage."

## 6. Type 6 — Analogy Overextension

**Mechanism**: an analogy deliberately maps ONE structural feature from
a source domain onto the target concept. If the mapping's boundary is
never stated, the learner silently imports ADDITIONAL features of the
source domain that were never intended to transfer — the analogy did its
job for the mapped feature and simultaneously installed a new error for
the unmapped ones.

**Diagnostic signature**: the specific wrong belief traces to a specific
analogy the learner was taught, and the error is a literal feature of
the SOURCE domain with no counterpart in the target concept. Asking
"where did that idea come from?" frequently surfaces the analogy by name
— this is the one birth type where the learner can often self-diagnose
the origin if asked directly, because the source domain is usually
something concrete and memorable.

**Cross-subject example**: the "water flowing through pipes" analogy for
electric current, taught to build intuition for current as something
that flows. Left unmapped, learners import the belief that current gets
"used up" or "absorbed" as it travels around a circuit (the way water
can be diverted or drained) — when current is actually conserved at
every point in a series circuit. The analogy correctly taught "flow"; it
silently also taught "flow can be consumed," which the source domain
happens to permit and the target concept does not.

**Repair implication**: this is the one type with a genuine prevention
mechanism at the moment of teaching — every concept entry's analogy
library (`../concepts/TEMPLATE.md`) must name the analogy's breaking
point THE FIRST TIME the analogy is used, not only after the
overextension has already occurred. Where it has already formed, the
repair names the unmapped feature explicitly ("the water analogy is
great for 'flow' — but current isn't like water in one specific way:
nothing gets used up") and treats it as a boundary lesson, not a full
retraction of the analogy.

## 7. Diagnostic procedure — identifying a new misconception's birth type

For a concept-entry author encountering a wrong answer pattern with no
existing misconception library entry yet, or a tutor observing a novel
systematic error live, work through these questions in order — the first
one that fires names the type:

```
1. Is the wrong answer the SAME wrong move every time, or does it vary
   unpredictably? If unpredictable → this is not a misconception at all;
   it is an absence of knowledge. Teach it; do not run a repair sequence.

2. Can the learner name where the idea came from, and does it trace to
   a specific analogy they were taught? → TYPE 6.

3. Does the error correlate specifically with the WRITTEN/symbolic form,
   and does it change or vanish when the same content is presented
   orally or with physical objects? → TYPE 4.

4. Does the error correlate with one specific WORD's everyday meaning,
   and does rephrasing the question without that word fix it? → TYPE 3.

5. Is the error near-universal across everyone taught by the same
   simplified early method, traceable to a narrow original example whose
   boundary was never named? → TYPE 5.

6. Can the learner state the wrong answer as an explicit RULE, and is
   that rule demonstrably TRUE in some other domain they already know?
   → TYPE 1.

7. If none of the above fired: is the error diffuse (no statable rule),
   resistant even under careful unhurried reasoning, and present before
   or independent of any formal instruction? → TYPE 2 — treat with the
   highest default regrowth-risk assumption
   (`../student-state/03 §3`'s prior) until evidence says otherwise.
```

This procedure is the tool a concept-entry author runs when writing a
new misconception library entry (`../concepts/TEMPLATE.md`'s misconception
section) for any of the ~1,800 concepts not yet covered — it replaces
improvising birth-type logic from scratch for every new entry.
