# Mastery Verification, Mastery Gates, and Adaptive Assessment

## 1. What mastery actually is

Mastery is six properties, each requiring its own kind of evidence. Missing
any one produces a characteristic later failure:

| Property | Evidence | If missing |
|---|---|---|
| **Correct** | production items | wrong answers (the visible failure) |
| **Fast** (fluent) | latency at or near the learner's fluent baseline | collapses under load when the concept becomes a *tool* inside bigger problems |
| **Confident** (calibrated) | confident-correct quadrant (04) | hedging, avoidance, knowledge invisible to learner and tutor |
| **Transferable** | new-surface and mixed items (see hierarchy below) | classroom-only knowledge, inert in the wild |
| **Durable** | delayed retrieval — next session and beyond | "knew it yesterday, gone today"; the same-session illusion |
| **Self-explained** | mechanism-level explanation (rubric, 10 §2) | right answers on procedure, helpless on any variation |

## 2. The evidence hierarchy

From weakest to strongest. Each level subsumes the ones before it; mastery
gates draw from the top half only.

```
 1. Recognition            ("which of these is it?")
 2. Cued recall            (prompted production)
 3. Production, taught surface
 4. Production, new numbers            ← weakest acceptable gate evidence
 5. Production, new surface story
 6. Production, new representation     (graph ↔ equation, words ↔ symbols)
 7. Mixed / unannounced                (item arrives with no topic label,
                                        among other topics — tests NOTICING,
                                        the true transfer bottleneck)
 8. Delayed                            (next session start or later —
                                        the only test of storage, not
                                        retrieval strength)
 9. Speeded, disguised                 (for repaired misconceptions:
                                        the old model rebids under time
                                        pressure — test where it rebids)
10. Spontaneous transfer               (learner applies it unprompted in a
                                        different context/subject — the
                                        strongest evidence that exists;
                                        never schedulable, always recordable)
11. Teach-back                         (learner teaches it, handles a
                                        planted confusion — rubric, 10 §5)
```

Levels 1–3 are *formative* instruments (purpose 4, 01 §1): they steer
teaching. They must never open gates — that is the recognition-inflation and
same-session-inflation failure mode (01 §4) built into policy.

## 3. Mastery gates

A gate is the decision point where the Brain certifies a concept and builds
on it. Requirements:

**The standard gate (concepts):**

- ≥2 production items on *unlike* new surfaces (levels 5–6), confident-correct;
- ≥1 mixed/unannounced item (level 7);
- ≥1 delayed retrieval success (level 8) — which means **no gate ever opens
  in the session where the concept was taught** (Universal Principle 17;
  consolidation is sleep-dependent);
- zero misconception-mapped distractors chosen across the gate set;
- latency consistent with fluency (against the learner's own baseline — see
  §5; absolute thresholds punish slow processors and reward the hasty);
- one mechanism-level explanation at rubric level ≥3 (10 §2).

**Additions by concept type:**

- **Tool skills** (number facts, decoding, notation): add an automaticity
  criterion — latency flatlined across a fluency burst. Tools that are
  correct-but-slow are not done: they will consume the working-memory slots
  that everything built on them needs (Delivery 2 §5).
- **Repaired misconceptions**: add the speeded, disguised, delayed re-probe
  (level 9) — plus the tier-2 "why is the wrong answer tempting?" item
  (03 §3). Weeds regrow where they were cut; the gate tests where they regrow.
- **Procedures**: gate items are *re-doings* (fresh execution), never
  descriptions ("what would you do?" is recognition of a procedure —
  worthless).

**Gate outcome discipline:**

- PASS → certify; enter the expanding review schedule (Delivery 2 §8);
  eligible as a prerequisite for dependents.
- FAIL → **never a restart.** Demote the review interval one step, run the
  failure autopsy (09), remediate the identified cause, re-gate with
  isomorphs after a new delay. A failed gate is scheduling information, not
  a verdict on the learner.
- The learner never sees gate machinery. Gates are the tutor's bookkeeping;
  the learner experiences only "we're moving on" or "let's make this one
  rock-solid first."

## 4. Adaptive assessment — choosing the next item

The principle behind all adaptive testing, stated without psychometrics:
**ask the question whose answer you can least predict.** An item the current
estimate says the learner will certainly pass (or certainly fail) carries
almost no information; the most informative item is the one where pass and
fail are both live possibilities.

Practical policy:

```
NEXT-ITEM SELECTION
├─ Last item confident-correct → step difficulty UP one notch
│  (or step transfer distance up — vary WHICH dial by what the
│  gate still lacks: fluency? distance? mixing?)
├─ Last item hesitant-correct → SAME difficulty, fresh isomorph
│  (consolidation, not advancement — 04 §5)
├─ Last item wrong via a MAPPED DISTRACTOR → stop measuring
│  difficulty; switch to the misconception's dedicated probe set
│  (the question has changed from "how much" to "which model")
├─ Last item wrong, blank/slow → step DOWN one notch; if two
│  consecutive, exit assessment into prerequisite micro-
│  diagnosis (02 §9)
└─ STOP when:
   • the pending decision (advance / consolidate / remediate /
     re-place) would not change with another item, OR
   • the session's affect budget is spent (2 visible failures), OR
   • fatigue signals appear (late-session data is corrupt anyway)
```

The stop rule is the heart of it. Assessment exists to make a decision;
measuring past the decision is spending learner trust on tutor curiosity.

## 5. Baselines are personal

Every fluency and latency judgment in this library is made against **the
learner's own baseline** — their typical response time on material they
demonstrably own — never against a population norm. A slow processor's
fluent is slower than a fast processor's struggling; absolute thresholds
systematically misclassify both. The baseline is recalibrated continuously
from the learner's confident-correct answers, per subject (processing speed
differs across domains — a fluent reader can be a slow calculator).
