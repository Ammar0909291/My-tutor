# Learning Trajectory

Where this learner is headed, how fast, and whether they are ready for
what's next — the model's time-derivative layer. Everything here is
per-domain (LAW 3): a global trajectory is a lie in detail, and the
learner accelerating in reading while plateaued in maths is the NORMAL
case, not an anomaly.

## 1. Mastery velocity

Concepts gated per unit of effort-time (sessions, not calendar — a
learner who comes twice a week isn't "slower" than a daily one), per
domain. Velocity's uses: honest forecasting ("two more weeks of this
and fractions close" — the visible horizon, `../decision-engine/07 §7`),
compaction triggers (sustained high velocity → offer the test-out), and
plan sizing. Velocity is NEVER surfaced as comparison (LAW 7) — it is
this learner against their own past only.

## 2. Acceleration — the truest teaching-fit signal

Velocity's trend. Rising velocity means the current approach FITS —
representations landing, floors solid, affect healthy; protect whatever
is working (change nothing gratuitously). Falling velocity is the
earliest warning the model has — earlier than failures, earlier than
affect events — and triggers a cause scan BEFORE anything breaks:
floors thinning (tool audit, 07 §4)? load creeping (concepts stacking
faster than automatization)? affect debt accumulating (06)? material
entering a genuinely harder region (normal — the KG's difficulty tags
say so; deceleration at a difficulty boundary is expected and not a
defect)?

## 3. Plateaus

Defined: N sessions without rung movement in a domain despite teaching
attempts. The diagnosis tree — because plateaus have four causes with
four different treatments, and thrashing is the classic mistreatment:

```
PLATEAU detected
├─ CONSOLIDATION? (reps accumulating, latency dropping, errors
│  shrinking — progress that doesn't move rungs yet)
│    → PATIENCE. This is the most common plateau and it is not a
│      problem; it is rung 4→5 happening invisibly. Do NOT change
│      methods — thrashing a consolidating learner interrupts the
│      very process that ends the plateau.
├─ METHOD MISMATCH? (escalations recurring, same-rung failures with
│  engaged affect)
│    → representation-family change (../decision-engine/05 §4's
│      stuck-concept protocol) + check history (08 §4) for what
│      HAS worked.
├─ FLOOR GAP? (errors systematic, tool audit shows sub-AUTOMATIC
│  tools, failures trace below the frontier)
│    → prerequisite work; the plateau is standing on sand.
└─ AFFECT DEBT? (velocity fell BEFORE the plateau; recovery events
   clustering; enjoyment sources gone quiet)
     → treat 06 first; no cognitive fix moves an unwilling learner.
```

## 4. Regression

Rung drops on previously verified territory. Route by shape: isolated
concept → memory event (07: FRAGILE/FORGOTTEN — scheduling handles it);
a CLUSTER regressing together → either their shared prerequisite
decayed (floor check) or a repaired misconception regrew and is
re-contaminating (03's regression events — check the ledger first when
the cluster shares a misconception territory); everything regressing →
not memory at all — affect, fatigue, or life outside the app (06 reads;
ask nothing intrusive, shrink demands, hold steady).

## 5. Momentum

The streak state: sessions ending on summits compound appetite; endings
on failure compound avoidance (why the close script and the summit
soft-trigger exist — `../decision-engine/07 §6`). The model tracks the
current run and spends it deliberately: momentum is WHEN to schedule
the feared thing, the re-gate, the misconception collision — hard
things land best on a three-win streak. Conversely, after a broken
streak, the next session buys the win back before attempting anything
expensive. Momentum is real, cheap to track, and almost never used by
AI tutors; human greats manage it instinctively.

## 6. Readiness

The forward-looking read the planner consumes before opening any new
concept: prerequisites' modulated rungs (02 §4 floor report) AND their
memory statuses (07 §4) AND the misconception ledger's territory check
(03 §4) AND the current affect trend (06) AND momentum (§5). Readiness
is a per-concept, per-moment derivation — never a stored score — and
its output is one of three: OPEN IT · CONSOLIDATE FIRST (named floors)
· NOT NOW (affect/momentum says buy a win first). The Decision Engine's
planner (07) asks; this model answers; and the answer always carries
its reasons (decision provenance, 10 §2).
