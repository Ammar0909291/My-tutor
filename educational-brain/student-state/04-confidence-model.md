# The Confidence Model

Confidence is stored separately from correctness, because the pair — not
either alone — is the teaching signal. The per-item reads and quadrant
responses live in `../assessment/04`; this file is the STANDING profile:
what this learner's confidence characteristically does, per domain, and
how it is evolving.

## 1. The five named patterns — from moment-reads to profile traits

Each pattern exists as a momentary read (assessment/04 handles the turn);
the model records when a pattern is CHRONIC — this learner's signature:

- **CORRECT-BUT-AFRAID** (chronic hesitant-correct with fear markers):
  knowledge outruns self-belief. Profile consequence: the
  show-them-their-record intervention (`../assessment/04 §4` — "when you
  say you're guessing, you're right 70% of the time") is scheduled,
  delivered, and its effect tracked. This single sentence, with real
  numbers, is the highest-ROI move this profile ever triggers — and the
  model must know whether it has been used, because it works best fresh.
- **CORRECT-BUT-GUESSING** (chronic disclosed guessing that scores above
  chance): either genuine partial knowledge the learner doesn't trust
  (treat as correct-but-afraid) or real guessing luck (small samples
  lie). The model distinguishes by domain breadth: luck is narrow,
  untrusted knowledge is broad.
- **WRONG-BUT-CONFIDENT** (chronic fast-wrong): the misconception
  signature — this trait is not stored here but routed: chronic
  confident-wrongness in a territory means ledger entries (03), and the
  profile records only the learner's general propensity to hold wrong
  models confidently (high propensity → probes arrive earlier in every
  new topic).
- **SLOW-BUT-CORRECT** (chronic effortful accuracy): consolidation as a
  standing style, or a deliberate temperament (05's overthinker). The
  model separates the two by trend: consolidation speeds up with reps;
  temperament doesn't. Temperamental slowness recalibrates every fluency
  judgment for this learner (`../assessment/05 §5` personal baselines —
  the profile is where those baselines live).
- **FAST-BUT-WRONG** (chronic rushing errors): usually 05's rusher, not
  a knowledge problem — the treatment is behavioural (committed pauses,
  prediction-first), and the model must NOT let rushing errors write
  low knowledge rungs (a rushed error is evidence about pace, not about
  the rung — LAW 1's evidence discipline).

## 2. The calibration profile

Per domain (LAW 3): direction (over/under-confident), magnitude, and
trend. Measured exactly as `../assessment/04 §3` specifies — this file
adds the standing uses:

- **Placement discount direction** (`../assessment/02 §5`): the intake
  self-report is discounted in the learner's recorded miscalibration
  direction, from session one of any NEW subject (the direction
  transfers across domains better than the magnitude).
- **Confidence-signal weighting**: for a chronically overconfident
  learner, spoken certainty is weak evidence (the classifier discounts
  it); for the underconfident, a hedge is weak evidence of a gap.
  Calibration data literally re-tunes the momentary classifier
  (`../decision-engine/02`) — climate correcting the weather report.
- **The calibration rubric level** (`../assessment/10 §7`): the
  learner's L0–L3 position, per domain, as a taught-skill target of its
  own — L3 (uses their own confidence as a tool) is a recorded mastery
  achievement with subject-wide consequences.

## 3. How confidence evolves — the asymmetry

Confidence builds and collapses on different curves, and the model
treats the asymmetry as law:

- **Builds slowly, linearly**: accumulated fluent successes, survived
  challenges, visible own-progress. No single event builds much — which
  is why manufactured praise does nothing (Delivery 2 §9: evidence
  moves self-belief; opinions don't).
- **Collapses steeply**: one public failure, one mocking moment, one
  session of stacked failures can undo weeks — steepest for fearful and
  perfectionist profiles. Consequence: the affect budget
  (`../assessment/02 §6`) is PERSONALIZED by this profile — a
  steep-collapse learner runs on a budget of one, permanently, until
  the profile itself heals.
- **Rebuilds along the repair path** (FEARFUL → FRAGILE → CONFIDENT,
  `../decision-engine/02 §3`) — in weeks, evidence-fed. The model
  records where on the path the learner stands PER DOMAIN, because the
  rebuild in mathematics does not transfer automatically to reading.
- **The target is calibration, not height.** The model never optimizes
  for maximum confidence — overconfidence is a failure state with its
  own costs (stops checking, resists repair, `../assessment/04 §3`).
  The mature endpoint: confidence that tracks competence, plus the
  learner's own working knowledge of their calibration (L3). A learner
  who says "I'm usually sure-and-right on these, but fractions fool
  me — let me check it" has the only confidence worth building.
