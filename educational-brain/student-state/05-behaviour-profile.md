# The Learning Behaviour Profile

How this learner works — pace, channels, attention, persistence,
curiosity — stored as **evidence-backed statistics, never identities**.
The profile's own founding caution: "learning styles" as fixed identities
do not survive testing (Delivery 2 §5's note). What DOES exist, and what
this profile records, is *what has actually worked for this learner, on
which kinds of content, how often* — action-outcome statistics that bias
selection (the affinity data `../decision-engine/04` filter 5 consumes)
without ever hardening into "she's a visual learner."

## 1. Pace style

- **THE RUSHER** — answers before the question ends; fast-wrong pattern
  without misconception structure (errors random, not systematic — the
  disambiguation from 03 territory). *Detection*: latency far below
  baseline on ERRORS specifically. *Consequence*: committed-pause
  protocols (prediction-first slows them for free — the commitment IS
  the brake); never treated as a knowledge problem (04 §1); praise
  attaches to caught-before-submitted self-corrections.
- **THE OVERTHINKER** — long latency on items they then get right;
  answer-changing away from correct first instincts. *Detection*:
  slow-correct with high first-instinct accuracy (track what they
  ALMOST said — voice self-talk reveals it). *Consequence*: the
  evidence intervention ("your first answer was right 85% of the time —
  data, not reassurance"); time-boxed items framed as games, never as
  pressure; distinguish from consolidation slowness by trend (04 §1).
- Most learners are neither, or are one only under specific states
  (rushing when bored, overthinking when afraid) — the profile records
  the CONDITION with the pattern, or it records noise.

## 2. Working habits (each: what it means, what it changes)

- **Reads aloud / self-talks** — a self-regulation ASSET, never
  suppressed; for the tutor it is free think-aloud protocol
  (`../assessment/06 §1`) — the profile flags it so the tutor
  deliberately leaves silence for it.
- **Skips instructions** — a design constraint, not a character flaw:
  this learner gets instructions IN the path (demo-first, worked
  example first, instructions revealed stepwise), never as preamble
  blocks they will skip.
- **Asks-before-trying vs tries-before-asking** — sets the scaffold
  default: the asker gets "try one first, then we talk"; the
  lone-struggler gets earlier check-ins (their "I'm stuck" arrives
  late or never — `../decision-engine/02 §4`'s trends must watch for
  silent struggling specifically for this profile).

## 3. Modality and action affinities

The statistics: per action family (Delivery 2 §6's six families) and
per knowledge type, the outcome history — what followed when this
family ran (success/struggle/recovery event). Consequences:

- **needs-demonstrations** (verbal explanations chronically bounce,
  demos chronically land): the selector's frame ordering flips —
  channel changes come one rung earlier on the escalation ladder
  (`../decision-engine/05 §1`).
- **analogies-land vs analogies-bounce**: some learner-concept pairs
  don't take analogies (`../decision-engine/05 §2` rung 3) — chronic
  bouncing generalizes to the learner level and biases the funnel
  toward direct mechanism + demonstration.
- **story-responder / experiment-responder / game-responder**: the
  positive affinities, used at tie-break strength (never overriding
  knowledge-type fit — a story-responder still gets worked examples
  for procedures; the story arrives as the anchor).
- **voice dependence**: learns better by ear (re-voicing text,
  retaining spoken > written) — sets the default assessment channel
  (`../assessment/06 §4` accommodation), pushes persistent visual
  anchors for anything multi-step (transience compensation), and
  raises the value of the voice channel's signals for this learner.
- Hardening rule: an affinity needs a **minimum evidence count across
  DIFFERENT concepts** before it biases selection (one great story
  lesson is a data point; five across topics is an affinity), and every
  affinity decays (LAW 2) — children especially outgrow them.

## 4. Attention span

Measured, not assumed: minutes-to-quality-decay per session, from the
fatigue trend signals (`../decision-engine/02` FATIGUED detection),
per time-of-day where visible. Consequences: the session time box is
PERSONAL (`../decision-engine/07 §1`'s scaling made concrete); the
core job is sized to fit inside the measured span, not the default;
decay onset triggers the summit-hunt (get the win in before the
window closes).

## 5. Persistence

How many unaided attempts before recovery utterances or abandonment —
this learner's struggle tolerance, per difficulty and per subject.
Consequences: step sizing (low persistence → smaller steps so failure
never stacks — `../decision-engine/07 §2` slow-down triggers arrive
earlier); scaffold fading speed (high persistence earns faster fades);
and the productive-struggle budget: how long the tutor lets them fight
before offering the ladder down (wait-time ceilings are persistence-
scaled — a gritty learner robbed of their fight resents the rescue;
a brittle one left fighting flees).

## 6. Curiosity style

- **Chain length** (Delivery 2 §3 by-personality): how many open loops
  this learner enjoys before anxiety — sets the close script's loop
  budget and mid-lesson detour policy.
- **Asker vs quiet wonderer**: the asker's questions arrive as gifts
  (`../decision-engine/06 §3` honored-hook rules); the quiet wonderer's
  curiosity shows as leaning-in prosody — the tutor voices the question
  FOR them ("you might be wondering why..."), which trains asking by
  modeling it.
- **What hooks them** (recorded per trigger type: surprise /
  contradiction / secret / personal stake / challenge): the anchor and
  curiosity-hook selection for every new concept reads this first.
