# ORGANIZE Family

Actions whose job is arranging or relating knowledge — sorting instances,
mapping relationships, translating between representations, or
scheduling when knowledge gets revisited — rather than teaching a single
new instance. ORGANIZE actions typically come AFTER the concept has some
initial foothold (from a SHOW, TELL, or DO action); they consolidate and
structure what is already partially there rather than installing it for
the first time.

## 20. Classification/Sorting

**What it is**: the learner sorts a set of instances into categories
defined by the concept — distinguishing examples from non-examples, or
sorting instances by which sub-type they belong to.

**Fits**: concept (`../decision-engine/04` filter 2's "concept →
analogy, contrasting cases, classification, drawing" row).

**Setup cost**: MEDIUM — requires a prepared set of instances that
includes genuine near-miss non-examples (items that share surface
features with the concept but fail the defining criterion), not just
obviously-different distractors.

**Procedure**: present the mixed set, ask the learner to sort, and treat
the near-misses as the actual teaching content — a learner who correctly
separates obviously-unlike items has learned nothing; correctly
separating near-misses is where the concept's boundary gets tested and
sharpened.

**Failure mode**: a sorting set with no genuine near-misses turns the
task into pattern-matching on surface features, which produces apparent
success with no boundary knowledge behind it.

## 21. Concept Map

**What it is**: a visual or verbal map of how multiple concepts relate
to each other — prerequisites, contrasts, hierarchies — rather than a
single concept in isolation.

**Fits**: concept.

**Setup cost**: HIGH — requires meta-language and abstraction the
learner may not have; `../decision-engine/04` filter 4 explicitly
excludes this action for young children ("concept maps... require
meta-language young children don't have (cut below ~10)").

**Procedure**: use only once at least two related concepts already have
some individual foothold — a concept map built on concepts that don't yet
exist for the learner has nothing to map. Most valuable during
consolidation and review, connecting a category the learner is finishing
to the category ahead (`../placement/05`'s progression engine visible-
horizon rule).

**Failure mode**: introducing a concept map as the FIRST exposure to a
new concept, before either related concept has any individual grounding
— this asks the learner to hold an abstraction about relationships
between two things they don't yet understand individually.

## 22. Matching/Translation

**What it is**: the learner translates the same underlying content
between two representational forms — equation to graph, graph to words,
words to table — in either direction.

**Fits**: concept, procedure. Named directly: "graph ↔ equation ↔ words
↔ table. Translation between representations is where understanding
lives" (`../assessment/06 §2`, cited "Delivery 2 §6, Matching").

**Setup cost**: MEDIUM.

**Procedure**: test BOTH directions explicitly, never just one. One-way
success with reverse failure is a precise, common, and diagnostic
finding — a learner who can plot a graph from a given equation but cannot
read the equation back off a given plot has memorized a procedure in one
direction without holding the underlying relationship that would make
the reverse trivial.

**Diagnostic value**: because Matching tests translation in both
directions by design, a directional asymmetry it reveals names exactly
what needs teaching next (`../assessment/06 §2`) — this is one of the
most efficient diagnostic actions in the catalog precisely because
failure has a specific, actionable shape rather than a generic "wrong."

## 23. Retrieval-Schedule Prompt

**What it is**: a short retrieval item deliberately spaced apart from
original teaching — not a new-content action but the action that keeps
already-taught content accessible over time, run at the interval the
memory engine's schedule specifies.

**Fits**: fact/convention (default reviewable type), procedure (tool
skills needing automaticity bursts).

**Setup cost**: LOW.

**Procedure**: cue with the encoding context rather than a bare recall
question — "remember the pizza-sharing approach?" rather than "what is
fraction equivalence?" (`../placement/06 §3`'s cue-design rule: cued,
context-rich retrieval beats bare recall for reactivating a fading
trace). This action is the operational form the memory engine's due-item
queue takes at the turn level — the schedule decides WHEN; this action
is HOW that moment is delivered.

**Relationship to OPENING**: this is the default action that populates a
session's OPENING phase (`../decision-engine/07 §1`) — due-review
retrieval runs as a sequence of Retrieval-Schedule Prompts, one per due
item, before any new-content action from another family begins.

**Failure mode**: phrasing the retrieval prompt as a yes/no memory
question ("do you remember X?") rather than a produce-the-answer item —
memory questions test meta-memory (does the learner believe they
remember), not the retrieval itself, and are explicitly banned by the
warm-up protocol this action shares its cue-design rule with
(`../placement/06 §3`: "do not ask 'do you remember X?'").
