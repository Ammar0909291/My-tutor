---

## Component 1 — Learning Objective

**Mastery statement:** A student who achieves mastery can (1) apply the 4-step pattern recognition process to identify a rule in a numerical, geometric, or logical sequence, (2) state the rule in both recursive and closed-form where possible, (3) verify the rule on unseen instances, (4) use the rule to predict terms well beyond the given examples, (5) recognise that the same finite set of initial terms may be generated by more than one rule, and (6) detect structural patterns across mathematical domains including combinatorics.

**NOT mastery:** A student who memorises the next term in a familiar sequence without stating the underlying rule, or who insists there can only be one correct pattern for a finite sequence, or who cannot detect patterns outside number lists, has NOT achieved mastery.

---

## Component 2 — Prerequisite Verification

**Hard prerequisites:** `math.found.mathematical-thinking` — student can reflect on what multiple instances have in common.

**Verification gate (TA-A01 opening):** Student can identify what is the same across three instances (e.g., "each group of dots is one larger than the last"). If not, route to `math.found.mathematical-thinking`.

---

## Component 3 — Misconception Registry

| ID | Label | Description | Type | MAMR Priority |
|----|-------|-------------|------|---------------|
| MC-1 | Pattern = memorisation | Student believes pattern recognition means recalling observed instances; does not seek an underlying structural rule | Foundational | 1st |
| MC-2 | One sequence, one pattern | Student insists any number sequence has exactly one correct pattern; rejects that multiple rules can generate the same initial terms | Secondary | 2nd |
| MC-3 | Patterns only in number lists | Student cannot detect patterns in geometric, logical, or combinatorial contexts | Secondary | 2nd (independent) |

**MAMR Rule:** MC-1 is foundational — a student who treats pattern recognition as memorisation will never extract a structural rule, making the deeper concepts of MC-2 and MC-3 repair inaccessible. Clear MC-1 first.

**TA-A01 — Concrete: Discovering Patterns in Objects**

*Primitive sequence:* `P02 → P03 → P26 → P06 → P55 → P49`

**P02 — CONTEXT ESTABLISHMENT:**
"Pattern recognition is one of the most powerful tools in mathematics. Let's begin by finding structural rules in something concrete and visual."

**P03 — CONCRETE MANIPULATION:**
Present three visual sequences:

```
Sequence A (dot rows):   1 dot, 2 dots, 3 dots, 4 dots, ?

Sequence B (square tiles):
  Row 1: □          (1 tile)
  Row 2: □ □        (3 tiles total)
  Row 3: □ □ □      (6 tiles total)
  Row 4: □ □ □ □    (10 tiles total)
  Row 5: ?

Sequence C (difference chain): 2, 5, 10, 17, 26, ?
```

"For each: what comes next? More importantly — WHAT IS THE RULE GENERATING THIS SEQUENCE?"

**P26 — WORKED EXAMPLE (Sequence A):**
```
Sequence A: 1, 2, 3, 4, ...

Step 1 (Observe): terms are 1, 2, 3, 4
Step 2 (Extract rule): each term is exactly 1 more than the previous → term n = n
Step 3 (Verify): n=3: rule gives 3 ✓; n=4: rule gives 4 ✓
Step 4 (Predict): n=10: term 10 = 10; n=100: term 100 = 100

KEY INSIGHT: The RULE "term n = n" is more powerful than the answer "5".
It predicts any term instantly without listing all terms before it.
```

**P06 — EXPLICIT STRATEGY INSTRUCTION:**
```
THE PATTERN RECOGNITION PROCESS (4 steps):
1. OBSERVE — list at least 4 terms and compute their consecutive differences.
2. EXTRACT THE RULE — identify the operation or property linking each term to its
   position n (closed-form) or to the previous term (recursive).
3. VERIFY — test the rule on at least 2 instances NOT used in step 2.
4. PREDICT — apply the rule to generate new terms or a general formula for term n.

The RULE is the goal. The given terms are only the raw material.
```

**P55 — COMPREHENSION CHECK:**
"Apply the 4-step process to Sequence B (1, 3, 6, 10, ...). What is the rule?"

*Expected: Differences: 3−1=2, 6−3=3, 10−6=4 → differences increase by 1 each time. Rule: term n = n(n+1)/2. Verify: n=3: 6 ✓, n=4: 10 ✓. Predict: n=5 = 15.*

**TA-A02 — Concrete: Recursive vs. Closed-Form Rules**

*Primitive sequence:* `P01 → P26 → P07 → P55 → P49`

**P01 — ATTENTION ANCHOR:**
"All patterns produce rules. But rules come in two structural types — and understanding the difference is essential for analysis."

**P26 — WORKED EXAMPLE:**
Two structural types illustrated with the sequence 3, 6, 12, 24, 48, ...:

```
TYPE 1 — RECURSIVE RULE: each term defined via the previous term
  a₁ = 3
  aₙ = 2 × aₙ₋₁   (multiply previous term by 2)
  To find term 10: compute a₂=6, a₃=12, a₄=24, ..., a₁₀=1536
  Limitation: to find term 100, you must compute all 99 terms before it.

TYPE 2 — CLOSED-FORM RULE: each term defined directly from its position n
  aₙ = 3 × 2ⁿ⁻¹
  To find term 10: 3 × 2⁹ = 3 × 512 = 1536  (one calculation)
  Power: term 100 = 3 × 2⁹⁹ — instant, regardless of all previous terms.
```

**P07 — PERCEPTUAL SALIENCE:**

| Property | Recursive Rule | Closed-Form Rule |
|---|---|---|
| Defines term n via | Previous term(s) | Position n directly |
| Compute term 100 | Need all 99 terms before | One calculation |
| Easier to discover | Often yes (see the step) | Sometimes requires algebra |
| Predicts any term | No (must step through) | Yes |

"In analysis, BOTH are useful. Closed-form is preferred when available because it answers: what is term n for ANY n — no stepping required."

**P55 — COMPREHENSION CHECK:**
"State both a recursive and a closed-form rule for 4, 8, 12, 16, 20, ... Compute term 50 using the closed-form."

*Expected: Recursive: a₁=4, aₙ=aₙ₋₁+4. Closed-form: aₙ=4n. Term 50 = 4×50 = 200.*

**TA-A03 — Pictorial: Geometric Pattern Recognition**

*Primitive sequence:* `P01 → P26 → P55 → P49`

"Patterns appear in shapes and geometric structures — not only in number sequences. The same 4-step process applies."

**P26 — WORKED EXAMPLE:**
Staircase pattern:

```
Step 1:  □                  (1 square)
Step 2:  □ □                (3 squares total)
         □
Step 3:  □ □ □              (6 squares total)
         □ □
         □
Step 4:  □ □ □ □            (10 squares total)
         □ □ □
         □ □
         □

Count: 1, 3, 6, 10, ...
```

Apply 4-step process:
```
Step 1 (Observe): 1, 3, 6, 10
Step 2 (Extract rule):
  Differences: 2, 3, 4 — each new step adds a column of n squares
  → at step n, add n squares to step n−1
  Recursive: a₁=1, aₙ=aₙ₋₁+n
  Closed-form: aₙ = n(n+1)/2  (sum 1+2+3+...+n)
Step 3 (Verify): n=4: 4×5/2=10 ✓; n=3: 3×4/2=6 ✓
Step 4 (Predict): n=10: 10×11/2=55 squares
```

"The GEOMETRIC staircase structure makes the rule visible: step n adds a column of n squares, so total = 1+2+...+n = n(n+1)/2."

"Why does seeing the geometric structure make it easier to discover n(n+1)/2 than looking at '1, 3, 6, 10' as bare numbers?"

*Expected: The visual shows EXACTLY what gets added at each step (a new column of n squares). The addition 1+2+3+...+n becomes physically visible in the staircase — no algebra required to see the pattern.*

**TA-A04 — Pictorial: Multiple Patterns from the Same Terms**

*Primitive sequence:* `P01 → P26 → P55 → P49`

**P01 — ATTENTION ANCHOR:**
"Critical analysis: the SAME finite sequence of terms can be generated by more than one rule. This challenges the assumption that there is only one 'correct' pattern."

**P26 — WORKED EXAMPLE:**
Given: 1, 2, 4 (only three terms shown).

```
RULE A (doubling):
  aₙ = 2ⁿ⁻¹
  n=1: 2⁰=1 ✓   n=2: 2¹=2 ✓   n=3: 2²=4 ✓
  Next term (n=4): 2³ = 8

RULE B (quadratic):
  aₙ = (n² − n + 2) / 2
  n=1: (1−1+2)/2 = 1 ✓
  n=2: (4−2+2)/2 = 2 ✓
  n=3: (9−3+2)/2 = 4 ✓
  Next term (n=4): (16−4+2)/2 = 7

RULE A predicts term 4 = 8.
RULE B predicts term 4 = 7.
BOTH RULES FIT THE GIVEN THREE TERMS EXACTLY.

Conclusion: With only three terms, we cannot determine which rule is "correct."
We need CONTEXT — what mathematical object generates this sequence — to decide.
```

**P55 — COMPREHENSION CHECK:**
"A student sees 1, 2, 4, 8 and says 'the next term is definitely 16.' What is wrong with the word 'definitely'?"

*Expected: Four terms support the doubling rule, but infinitely many other rules also fit four terms. Without knowing the mathematical source of the sequence, 'definitely' is unjustified. We can say the doubling rule fits, but other rules also fit.*

**TA-A05 — Abstract: Algebraic Pattern Recognition**

*Primitive sequence:* `P01 → P26 → P55 → P49`

**P01 — ATTENTION ANCHOR:**
"We've recognised patterns in concrete objects, geometric shapes, and number sequences. Now: patterns expressed as algebraic identities — where the structure appears in symbolic form."

**P26 — WORKED EXAMPLE:**
Pascal's Triangle row sums:

```
Row 0:     1            → sum = 1
Row 1:    1 1           → sum = 2
Row 2:   1 2 1          → sum = 4
Row 3:  1 3 3 1         → sum = 8
Row 4: 1 4 6 4 1        → sum = 16

Pattern in sums: 1, 2, 4, 8, 16 → sums = 2ⁿ for Row n

Step 2 (Rule): Row n of Pascal's Triangle sums to 2ⁿ.
Step 3 (Verify): Row 5 = 1+5+10+10+5+1 = 32 = 2⁵ ✓
Step 4 (Predict): Row 10 sums to 2¹⁰ = 1024.
```

"The ALGEBRAIC IDENTITY: Σ C(n,k) = 2ⁿ for k=0 to n. This is a deep structural pattern connecting Pascal's Triangle to powers of 2 — discovered by finding the pattern and then PROVING it holds for all n."

**P55 — COMPREHENSION CHECK:**
"Why does recognising the pattern (sums are powers of 2) alone NOT constitute a mathematical proof? What additional step is required?"

*Expected: Pattern recognition identifies the rule and verifies it on specific cases. Proof (a separate act) demonstrates why the rule MUST hold for ALL n — not just the observed cases. Pattern recognition is the discovery stage; proof is the justification stage.*

**TA-A06 — Abstract: Cross-Domain Pattern Recognition (Combinatorics)**

*Primitive sequence:* `P01 → P26 → P55 → P49`

**P01 — ATTENTION ANCHOR:**
"Patterns cross mathematical domain boundaries. The same structural rule appears in counting, geometry, algebra — often unexpectedly. Recognising cross-domain patterns is a mark of analytical maturity."

**P26 — WORKED EXAMPLE:**
The sequence 1, 1, 2, 3, 5, 8, 13, 21, ... (Fibonacci) appears in three unrelated domains:

```
DOMAIN 1 (Counting rabbit pairs — original Fibonacci problem):
  f(1)=1 pair, f(2)=1 pair, f(n)=f(n−1)+f(n−2) — same recursive rule.

DOMAIN 2 (Tile patterns — how many ways to tile a 1×n strip with 1×1 and 1×2 tiles):
  1×1 strip: 1 way   → 1
  1×2 strip: 2 ways  → but Fibonacci here starts 1,1 not 1,2... Let me use:
  1×n tiling: T(n) = T(n−1) + T(n−2) — same Fibonacci recurrence!
  T(1)=1, T(2)=2, T(3)=3, T(4)=5, T(5)=8 — these ARE Fibonacci numbers.

DOMAIN 3 (Golden ratio — geometric proportion):
  As n→∞, F(n+1)/F(n) → φ = (1+√5)/2 ≈ 1.618 — appears in spiral geometry.
```

"The Fibonacci recurrence F(n)=F(n−1)+F(n−2) is the pattern. It generates the same sequence regardless of whether you are counting rabbits, tiling strips, or studying geometric spirals."

*P76 Cross-Link Bridge: math.disc.combinatorics — combinatorial structures (tiling, counting sequences, Pascal's Triangle) are a primary domain where pattern recognition is the fundamental tool. Patterns in counting objects often reveal recursive structures identical to those in number theory and geometry. When students study combinatorics, they will find the same patterns they learned here appearing as combinatorial identities.*

**P55 — COMPREHENSION CHECK:**
"Why does the SAME recurrence F(n)=F(n−1)+F(n−2) appear in such different domains (rabbits, tiles, geometry)? What does this tell you about mathematical patterns?"

*Expected: Mathematical patterns are structural — they are properties of the RULE itself, not of the physical domain. Wherever the recursive structure 'count = sum of two previous counts' applies, Fibonacci numbers emerge. This shows patterns are abstract structures, not domain-specific observations.*

**TA-A07 — Mastery Gate**

*Primitive sequence:* `P01 → P91`

**P91 expansion:** `P77 → P55 → P76 → P55 → P75 → P55 → P74 → P55 → P78`

**P01 — ATTENTION ANCHOR:** "All pattern recognition skills developed. Confirming mastery."

**P77 — MASTERY PROBE:**
"State the 4-step pattern recognition process. Distinguish recursive from closed-form rules. Explain why a finite sequence does not determine a unique pattern."

**P55 — COMPREHENSION CHECK:** "When is a recursive rule more useful than a closed-form? When is closed-form preferred?"

*Expected: Recursive is easier to discover (just look at consecutive differences/ratios). Closed-form is preferred for large-n predictions (no stepping through all previous terms) and for algebraic analysis.*

**P76 — TRANSFER PROBE:**
"In music, a composer notices: bar 1 has 1 note, bar 2 has 2, bar 3 has 4, bar 4 has 8. (a) State the rule. (b) How many notes in bar 10? (c) Could a different rule also generate 1, 2, 4, 8? Give one alternative."

*P76 note: cross_links=[math.disc.combinatorics]; this probe uses musical context for self-contained real-world transfer and a combinatorial alternative for the cross-link. The connection to combinatorics is structural — many combinatorial counting problems generate the same exponential patterns.*

**P55 — COMPREHENSION CHECK:** "Why does the doubling rule fail to tell us what comes after bar 4 without additional context about the composer's intention?"

*Expected: Multiple rules generate 1, 2, 4, 8. Without knowing whether the composer chose doubling specifically or was following another structure, the next term is underdetermined.*

**P75 — MISCONCEPTION PROBE:**
"A student says: 'I've seen the sequence 1, 4, 9, 16 before — I know the rule is n² without applying the 4-step process, so I can skip it.' What risk does this shortcut create?"

*Expected: MC-1. Recognising a familiar sequence by memory does NOT identify the rule — it bypasses rule extraction. The student might apply the wrong rule when the sequence appears in an unfamiliar context. Also: 1, 4, 9, 16 could be generated by other rules (e.g., aₙ = n⁴−6n³+14n²−12n+4 for these four terms). Memory ≠ pattern recognition.*

**P55 — COMPREHENSION CHECK:** "Give one case where a familiar sequence appearing in an unfamiliar context would lead to the wrong rule if you relied on memory alone."

*Expected: E.g., 1, 4, 9, 16 appears in a combinatorial problem where it actually represents (C(n+1,2))² — the rule is n² here, but the structural reason is combinatorial, not purely arithmetic. Understanding WHY prevents misapplication.*

**P74 — APPLICATION PROBE:**
"Identify the pattern in this table and state a general formula:

| n | Triangle area |
|---|---|
| 1 | 1 |
| 2 | 4 |
| 3 | 9 |
