## Component 3 — Core Explanation

**A UFD is defined by the unique-factorization property itself**: an integral domain $R$ is a Unique Factorization Domain if every nonzero, non-unit element factors into irreducibles, and this factorization is UNIQUE up to reordering the factors and multiplying by units (invertible elements). This directly generalizes the familiar integer case: $\mathbb{Z}$'s elements factor into primes uniquely up to order and sign ($\pm1$ being $\mathbb{Z}$'s units) — `math.nt.fundamental-theorem-arithmetic`'s own statement, restated in this concept's general ring-theoretic language.

**Every PID is a UFD — `math.abst.pid`'s own structural facts are the proof mechanism**: `math.abst.pid` already establishes that in a PID, every prime ideal is maximal and irreducibles equal primes. These facts are PRECISELY what makes the unique-factorization argument work: existence of a factorization into irreducibles follows from a PID's ascending-chain condition on ideals (no infinite strictly increasing chain of principal ideals, ruling out infinite factorization), and UNIQUENESS follows from irreducibles being prime (a prime element dividing a product must divide one of the factors — the key lemma that forces any two factorizations to match up term by term). So "every PID is a UFD" is not an independent new fact requiring separate machinery — it is a direct consequence of structure `math.abst.pid` already proved.

**UFD is a strictly BROADER class — $\mathbb{Z}[x]$ is the standard separating example**: the converse of "every PID is a UFD" is FALSE — there exist UFDs that are not PIDs. The standard example is $\mathbb{Z}[x]$ (polynomials with integer coefficients): it IS a UFD (a consequence of Gauss's Lemma, which lifts unique factorization from $\mathbb{Z}$ and from $\mathbb{Q}[x]$ up to $\mathbb{Z}[x]$ itself, not derived in full here) but is NOT a PID — the ideal $(2,x)$ (all polynomials with even constant term) cannot be generated by a single element, so it fails to be principal. This single example shows the UFD property is strictly WEAKER than (implied by, but not equivalent to) the PID property.

## Component 4 — Worked Examples

**Example 1 (LO1 — restating FTA in UFD language, breaking MC-1)**: `math.nt.fundamental-theorem-arithmetic`'s own statement, "every integer $>1$ factors uniquely (up to order) into primes," restated in THIS concept's terms: $\mathbb{Z}$ is an integral domain; its irreducible elements are EXACTLY the primes (and their negatives, since $-1$ is a unit); every nonzero, non-unit integer factors into these irreducibles UNIQUELY up to reordering AND up to multiplication by units ($\pm1$) — e.g. $12=2\times2\times3=(-2)\times(-2)\times3$, both valid factorizations differing only by unit factors. This is EXACTLY "$\mathbb{Z}$ is a UFD," confirming LO1's identification concretely and resolving the forward reference `math.nt.fundamental-theorem-arithmetic` itself explicitly left open pending this concept's authoring.

**Example 2 (LO2 — a PID automatically inheriting UFD status, breaking MC-2)**: `math.abst.pid`'s own worked example already confirms $\mathbb{Z}$ is a PID (every ideal is $(n)$ for some $n$) with irreducibles equal to primes (verified there for $7$) and every prime ideal maximal. By THIS concept's theorem ("every PID is a UFD"), $\mathbb{Z}$'s UFD status follows IMMEDIATELY from its already-established PID status — no separate factorization-uniqueness proof is needed; the PID structure `math.abst.pid` already verified IS the proof. Contrast: verifying UFD status "from scratch" (directly checking factorization uniqueness for every element) would be far more work than simply invoking "PID $\Rightarrow$ UFD" once PID status is known.

**Example 3 (LO3 — $\mathbb{Z}[x]$ as a UFD that is not a PID, breaking MC-3)**: checking the ideal $I=(2,x)=\{2p(x)+xq(x):p,q\in\mathbb{Z}[x]\}$ in $\mathbb{Z}[x]$ — this is exactly the set of polynomials with EVEN constant term. Suppose (for contradiction) $I=(g(x))$ for a single generator $g$. Since $2\in I$, $g$ must divide $2$ in $\mathbb{Z}[x]$, forcing $g$ to be a constant ($\pm1$ or $\pm2$); since $x\in I$, $g$ must divide $x$ too, forcing $g=\pm1$ — but then $I=(\pm1)=\mathbb{Z}[x]$ itself, contradicting that $I$ excludes polynomials with ODD constant term (like $1\notin I$). So $I$ is NOT principal, confirming $\mathbb{Z}[x]$ is NOT a PID — while $\mathbb{Z}[x]$ nonetheless IS a UFD (via Gauss's Lemma, stated not derived here), directly confirming LO3's claimed strict separation between the two classes.

## Component 5 — Teaching Actions

### Teaching Action A01 — The Fundamental Theorem of Arithmetic IS "$\mathbb{Z}$ Is a UFD" (Primitive P11: Representation Shift)

State: "`math.nt.fundamental-theorem-arithmetic`'s statement about integers factoring uniquely into primes isn't merely SIMILAR to the UFD definition — restated in this concept's language, it IS exactly the claim that $\mathbb{Z}$ is a UFD." Walk Example 1's direct restatement.

- **MC-1 hook**: ask "is the Fundamental Theorem of Arithmetic a separate fact merely analogous to the UFD definition, or literally an instance of it applied to $\mathbb{Z}$?" — a "merely analogous" answer reveals MC-1 (missing the direct, exact identification).

### Teaching Action A02 — PID Status Already Proves UFD Status — No Separate Verification Needed (Primitive P28: Conflict Evidence)

Present Example 2's direct evidence: `math.abst.pid`'s own already-established PID structure for $\mathbb{Z}$ is IMMEDIATELY sufficient to conclude UFD status, via the "every PID is a UFD" theorem, with zero additional factorization-uniqueness work. State: "once you know a ring is a PID — using `math.abst.pid`'s own structural facts (irreducibles equal primes, prime ideals maximal) — UFD status comes for free; you never need to separately re-verify factorization uniqueness from scratch."

- **MC-2 hook**: ask "if a ring is already known to be a PID, does confirming it is also a UFD require a separate, independent factorization-uniqueness verification?" — a "yes, separate verification needed" answer reveals MC-2 (missing that PID status already guarantees UFD status via the direct theorem).

### Teaching Action A03 — UFD Is Strictly Broader Than PID — $\mathbb{Z}[x]$ Proves It (Primitive P06: Contrast Pair)

Contrast the tempting assumption "PID and UFD are basically the same class" against Example 3's fully worked demonstration that $\mathbb{Z}[x]$ genuinely fails to be a PID (the ideal $(2,x)$ is provably non-principal) while still being a UFD. State: "'every PID is a UFD' only runs in ONE direction — $\mathbb{Z}[x]$ is the standard example proving the converse fails: a ring can have unique factorization without every ideal being principal."

- **MC-3 hook**: ask "does 'every PID is a UFD' mean the two classes of rings are essentially the same, or can a ring be a UFD without being a PID?" — an "essentially the same" answer reveals MC-3 (missing the genuine strict separation $\mathbb{Z}[x]$ demonstrates).
