**Core idea:** Two fractions a/b and c/d are equivalent if they represent the same quantity; this holds exactly when a×d = b×c, and is generated by multiplying (or dividing) both numerator and denominator by the same nonzero integer. Equivalence is a multiplicative relationship, not an additive one.

**Key vocabulary:** equivalent, numerator, denominator, cross-multiplication, simplify, multiplier.

**Why difficulty = 2:** Students must coordinate two abstraction levels (concrete equal-parts and formal multiplicative rule) and resist a persistent additive misconception. The concept is bounded in scope but non-trivial in its conceptual demand.

---

## 1. Learning Objective

A student who achieves mastery demonstrates that two fractions are equivalent when multiplying both numerator and denominator by the same nonzero integer, AND can generate equivalent fractions with a specified denominator, AND can verify equivalence using cross-multiplication. NOT merely recognising that "1/2 and 2/4 look related" or listing equivalents by repeated doubling. A student who generates equivalents mechanically (×2 only, or by guess) but cannot explain why the value is unchanged, or who uses additive reasoning (adds the same number to top and bottom), has NOT achieved mastery.

**Accuracy threshold:** 85% across generation, verification, and classification tasks.

**TA-C02 — MC-1 Repair: Adding Does Not Preserve Value**

Primitive sequence: `P01 → P27 → P28 → P29 → P55 → P30 → P31 → P34 → P55 → P49 | P50 → P32`

`P01` target_element: the claim "add 1 to top and bottom to get an equivalent"  
`P27` schema exposure: "You added 1 to both numerator and denominator to get 2/4 from 1/3. Let's test whether 1/3 = 2/4."  
`P28` cognitive conflict: "As decimals: 1 ÷ 3 ≈ 0.333, but 2 ÷ 4 = 0.500. These are different values. The 'add 1' rule gives a different fraction, not an equivalent one."

`P31` schema replacement: "To generate an equivalent fraction: MULTIPLY both numerator and denominator by the same nonzero integer. Never add."  
`P34` closed: "Using multiplication only, find a fraction equivalent to 3/7 with denominator 21."  
`P55` wait: 5 s

**TA-C03 — MC-2 Repair: Non-Simplified Fractions Are Valid**

Prerequisite: MC-1 cleared (or MC-2 active independently).

Primitive sequence: `P01 → P27 → P28 → P30 → P31 → P33 → P34 → P55 → P49 | P50`

`P01` target_element: the pair 3/4 and 6/8  
`P27` schema exposure: "You said 6/8 isn't equivalent to 3/4 because it's not simplified. Let's apply the definition."

## 6. Misconception Engine

---

### MC-1 — Adding Same Number to Numerator and Denominator Creates Equivalents

**Observable symptom:** Student generates 2/4 from 1/3 by adding 1 to both, or generates 4/7 from 3/6 by adding 1. States "add the same number to top and bottom."

**Diagnostic trigger:** P41 after "make a fraction equivalent to 1/3" (DB-2; TA-C01).

**Root cause:** Symmetry heuristic — students correctly apply "same operation to both sides" from equation-solving but conflate multiplication and addition as equally valid symmetric operations on fractions.

**Repair chain:** P27 (expose additive belief) → P28 (conflict: 1/3 ≈ 0.333 ≠ 2/4 = 0.500) → P29 (pause) → P30 (bridge: adding changes ratio; multiplying scales both parts together) → P31 (install correct rule: multiply) → P32 (three consolidation problems)

**S6-safe path:** Omit P28; route through P30 directly with concrete strips showing unequal coverage when addition is applied.

**MAMR status:** FOUNDATIONAL for MC-2. MC-1 must be cleared before MC-2 repair begins. Rationale: MC-2 repair involves cross-multiplication on non-simplified fractions; students carrying MC-1 will generate wrong "equivalents" to compare, making MC-2 repair incoherent.

---

### MC-2 — Only Simplified Fractions Are Valid / Equivalent

**Observable symptom:** Student accepts 3/4 as the "only real" fraction for that value; rejects 6/8 as "not a proper fraction" or "a different fraction" because it is not in lowest terms.

**Diagnostic trigger:** P41 "Is 6/8 equivalent to 3/4?" (DB-2; TA-C01).

**Root cause:** School instruction emphasises "always simplify your answer," which students interpret as: the simplified form is the fraction; all other forms are incorrect or inferior.

**Repair chain:** P27 (expose belief) → P28 (conflict: cross-multiplication confirms 6/8 = 3/4 by definition) → P30 (bridge: simplification finds the simplest name; all names for the same value are valid) → P31 (install: equivalent fractions are multiple names for one quantity) → P33 (discrimination training: {(3/4, 6/8) → equivalent} vs {(3/4, 5/8) → not equivalent})

**S6-safe path:** Omit P28; use P07 (number line) to show 6/8 and 3/4 land at the same point before introducing P30.

**MAMR status:** Independent of other misconceptions.
