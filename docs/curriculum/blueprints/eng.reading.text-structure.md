## Component 1 — Misconception Register

### MC-TEXT-STRUCTURE-IS-JUST-ABOUT-PARAGRAPH-BREAKS
- **Trigger signal:** Student treats "text structure" as a synonym for physical paragraph breaks (where the writer chose to start a new paragraph) rather than the underlying organizational PATTERN of ideas — sequence, cause-effect, compare-contrast, problem-solution, or description — that a passage follows, which is a logical relationship, not a formatting choice.
- **Conflict evidence [P28]:** Present a single unbroken paragraph that clearly follows a cause-effect pattern (e.g., "Heavy rain fell for three days, so the river rose above its banks, which flooded the nearby farms, destroying that season's crops.") and ask the student to identify the structure. If they say "there's no structure, it's just one paragraph," push: does this passage show one event happening and just describe it (description), or does it show one thing leading to another leading to another (cause-effect)?
- **Bridge text [P30]:** "Paragraph breaks are about formatting on the page; text structure is about how the IDEAS are connected underneath — do they happen in time order, does one cause the next, are two things being compared, is a problem being solved? A single paragraph, or even a single sentence, can still follow one of these patterns; a five-paragraph essay could still just be doing 'description' the whole way through."
- **Replacement text [P31]:** "Ignore where the paragraph breaks are for a moment. Ask instead: what's the logical relationship between the ideas? Time order → sequence. One thing leads to another → cause-effect. Two things set side by side → compare-contrast. A difficulty and a fix → problem-solution. Just describing features/qualities → description."
- **Discrimination pairs [P33]:** (a) Structure-as-formatting (paragraph breaks themselves define the structure) vs. (b) Structure-as-logical-pattern (the underlying relationship between ideas — sequence/cause-effect/compare-contrast/problem-solution/description — defines the structure, independent of paragraphing).
- **s6_path:** If detected, drop to PD-1 re-check, then re-run TA-1 with the single-paragraph cause-effect example before continuing.

### MC-A-TEXT-CAN-ONLY-HAVE-ONE-STRUCTURE-PATTERN
- **Trigger signal:** Student assumes an entire passage must follow exactly one structural pattern throughout and forces a single label onto a passage that actually shifts patterns (e.g., a passage that describes a problem using cause-effect reasoning, then shifts to problem-solution structure for the second half) — refusing to recognize that longer or more complex texts commonly combine multiple patterns in different sections.
- **Conflict evidence [P28]:** Present a two-paragraph passage where paragraph 1 explains why a lake became polluted (cause-effect: factory runoff caused fish die-offs) and paragraph 2 describes three different cleanup solutions being tried (problem-solution). Ask the student to name "the" structure of the whole passage, then ask whether paragraph 1 and paragraph 2 are doing the same organizational job.
- **Bridge text [P30]:** "Short, simple passages often use just one pattern all the way through — but longer or more complex texts frequently combine patterns, with different sections doing different organizational jobs. It's completely normal and correct to say 'this passage uses cause-effect in the first part and problem-solution in the second' rather than forcing one single label onto the whole thing."
- **Replacement text [P31]:** "Before locking in one structure label for an entire passage, check each section separately. If different sections show different relationships between ideas, name each section's pattern rather than picking just one overall label that doesn't really fit every part."
- **Discrimination pairs [P33]:** (a) Single-pattern-only (a text must be classified under exactly one structural label overall) vs. (b) Multi-pattern-recognition (different sections of a longer text can legitimately use different structural patterns, and naming each section's pattern separately is more accurate than forcing one label).
- **s6_path:** If detected, drop to PD-1, then re-present the Concrete Anchor's multi-room-house framing before continuing to TA-4.

## Component 2 — Prerequisite Diagnostic Block

### PD-1: Main Idea and Detail-Relationship Readiness
- **Tests:** Can the student identify how supporting details relate to a main idea — since text structure is essentially the pattern by which details are organized in relation to each other and to the main idea?
- **Probe:** Give a short paragraph and ask the student to state the main idea and then describe, in their own words, how the details are arranged (in order, compared, causally linked, etc.) without yet using formal structure vocabulary.
- **Pass condition:** Correctly states the main idea and gives a reasonable informal description of how the details relate.
- **Fail routing:** If the student cannot describe detail relationships at all, route to `eng.reading.main-idea-and-details` review — text-structure vocabulary is only useful once the student can already notice how details connect to each other.

## Component 3 — Concrete Anchor [P06]

**"The Five Blueprint Shapes."** Present five simple physical/drawn shape cards, each representing one structural pattern: (1) an ARROW CHAIN (→→→) for sequence/time-order; (2) a DOMINO CHAIN (one domino tipping the next) for cause-effect; (3) a SIDE-BY-SIDE SCALE (two pans balancing) for compare-contrast; (4) a BROKEN-THEN-FIXED GEAR icon for problem-solution; (5) a SNAPSHOT/CAMERA icon for description (just capturing features, no chain of events). For each shape, read a one-sentence example passage aloud and have the student pick the matching shape card. Then introduce a "multi-room house" blueprint: a floor plan with two rooms, one shaped like the domino chain and one shaped like the broken-then-fixed gear, showing that one "house" (one passage) can have different-shaped "rooms" (sections) — directly seeding MC-A-TEXT-CAN-ONLY-HAVE-ONE-STRUCTURE-PATTERN's fix. Emphasize: always check the shape section-by-section before assuming the whole house is one shape.

## Component 4 — Conceptual Development Sequence

### TA-1: Single-Paragraph Pattern Identification
Present 5 single-paragraph passages, one per structural pattern (sequence, cause-effect, compare-contrast, problem-solution, description), unbroken into multiple paragraphs, and have the student identify each pattern using the shape cards — directly countering MC-TEXT-STRUCTURE-IS-JUST-ABOUT-PARAGRAPH-BREAKS by proving pattern identification works independent of paragraph count.

### TA-2: Signal Words per Pattern
Teach the characteristic signal words for each pattern: sequence ("first, next, then, finally"), cause-effect ("because, as a result, therefore, so"), compare-contrast ("similarly, in contrast, both, whereas"), problem-solution ("the problem was, to fix this, one solution"), description ("for example, characteristics include, appears to be"). Student highlights signal words in the TA-1 passages and matches them to patterns.

### TA-3: Distinguishing Similar Patterns
Address common confusions: sequence vs. cause-effect (sequence is just time order with no causal claim; cause-effect specifically claims one event produced the next) and compare-contrast vs. description (compare-contrast requires two or more subjects being set against each other; description covers just one subject's features). Student practices distinguishing close pairs.

### TA-4: Multi-Pattern Passages
Present a 2-3 section passage where different sections use different patterns (as in the Conflict Evidence example) and have the student identify each section's pattern separately rather than forcing one overall label — directly countering MC-A-TEXT-CAN-ONLY-HAVE-ONE-STRUCTURE-PATTERN.

### TA-5: Constructing a Passage to a Specified Structure
Student writes an original short passage following an assigned structural pattern (e.g., "write a 4-sentence problem-solution passage about a school issue"), then swaps with a self-check against the signal-words and shape-card criteria — the generative capstone.

## Component 5 — Worked Examples

### WE-1: Five-Pattern Identification (Scaffolded)
**Passages:** "First, the seeds were planted. Next, they were watered daily. Then, sprouts appeared after a week. Finally, the plants were ready for harvest by autumn." (sequence) / "The bridge was closed for repairs, so traffic backed up for miles." (cause-effect) / "Cats are independent and often aloof, while dogs are typically social and eager to please." (compare-contrast) / "Local pollution had killed most of the fish, so the town built a new water treatment plant to restore the river." (problem-solution) / "The old lighthouse stands forty feet tall, painted red and white, with a rusted iron railing circling its top." (description)
**Walkthrough:** Match each passage to its shape card using both the logical relationship and the signal words present.

### WE-2: Distinguishing Sequence from Cause-Effect (Guided)
**Pair:** "She woke up, brushed her teeth, and ate breakfast." (sequence — no claim that one action caused the next, just time order) vs. "She overslept, so she skipped breakfast and rushed out the door." (cause-effect — oversleeping directly causes the skipped breakfast and rushing).
**Walkthrough:** Ask whether removing the causal signal word ("so") would change the passage's claim. In the sequence example, removing time-order words doesn't create a causal claim; in the cause-effect example, "so" is doing real logical work, not just marking order.

### WE-3: Multi-Pattern Passage (Independent)
**Passage (2 sections):** Section 1: "The old bridge had been closed for six months after inspectors found cracks in its support beams, forcing a ten-mile detour for daily commuters." (cause-effect) Section 2: "The city considered three options: repairing the original beams, building a temporary bypass, or constructing an entirely new bridge. After public hearings, they chose the temporary bypass as the fastest solution." (problem-solution)
**Walkthrough:** Student identifies Section 1 as cause-effect and Section 2 as problem-solution, explicitly stating that the passage as a whole uses two different patterns rather than forcing one label.

## Component 6 — Mastery Probe Set

**MP-1 [P36] — Single-paragraph cause-effect identification**
Given a single, unbroken paragraph following a cause-effect pattern, student correctly identifies the pattern and explains that paragraph count doesn't determine structure. *(Directly targets MC-TEXT-STRUCTURE-IS-JUST-ABOUT-PARAGRAPH-BREAKS.)*

**MP-2 [P36] — Multi-pattern passage identification**
Given a 2-section passage using two different structural patterns, student identifies each section's pattern separately rather than forcing one overall label. *(Directly targets MC-A-TEXT-CAN-ONLY-HAVE-ONE-STRUCTURE-PATTERN.)*

**MP-3 [P36] — Signal word matching**
Given 5 unlabeled passages, student identifies the structural pattern of each using signal words as evidence.

**MP-4 [P36] — Distinguishing close pairs**
Given a sequence/cause-effect pair and a compare-contrast/description pair, student correctly distinguishes each and explains the key difference.

**MP-5 [P36] — Original passage construction**
Given an assigned structural pattern, student writes an original passage following that pattern and self-verifies it against the signal-words criteria.

**Protocol routing paragraph:** At session open (S0), confirm PD-1 passes — text-structure vocabulary is only meaningful once the student can already notice how details relate to each other and to a main idea, since the five patterns are just names for detail-relationship types. During instruction (S1), watch for both misconception patterns: a student anchoring structure judgments to paragraph breaks (MC-TEXT-STRUCTURE-IS-JUST-ABOUT-PARAGRAPH-BREAKS) or forcing a single label onto a mixed-pattern passage (MC-A-TEXT-CAN-ONLY-HAVE-ONE-STRUCTURE-PATTERN) is skipping the actual logical-relationship analysis in favor of a surface shortcut — both should be redirected to the Five Blueprint Shapes anchor's section-by-section check. If the student cannot distinguish sequence from cause-effect during TA-3 (S6), drop back to WE-2's signal-word-removal test before returning to mixed practice. At mastery-probe stage (S9), do not certify mastery unless MP-1 and MP-2 both pass — a student who identifies single-pattern passages correctly (MP-3) but cannot handle a multi-pattern passage (MP-2) has not yet learned the full flexibility this concept requires for longer real-world texts.

## Component 8 — Adaptive Flags

- If the student anchors structure judgments to paragraph breaks, weight remediation toward MC-TEXT-STRUCTURE-IS-JUST-ABOUT-PARAGRAPH-BREAKS's bridge text and re-run TA-1 with single-paragraph examples of each pattern before returning to multi-paragraph texts.
- If the student forces one structural label onto mixed-pattern passages, weight remediation toward MC-A-TEXT-CAN-ONLY-HAVE-ONE-STRUCTURE-PATTERN's bridge text and TA-4's section-by-section practice specifically.
- This concept feeds two direct downstream unlocks (`eng.reading.genre-recognition`, `eng.reading.compare-and-contrast-texts`) — do not certify mastery without TA-4/MP-2's multi-pattern recognition, since `eng.reading.compare-and-contrast-texts` specifically extends the compare-contrast pattern to comparing entire texts against each other, which assumes the student already handles pattern identification flexibly rather than rigidly.
- Reuse the "five blueprint shapes" framing explicitly when introducing `eng.reading.genre-recognition` — genre conventions often correlate with characteristic structural patterns (e.g., persuasive texts frequently use problem-solution), giving the student a bridge from structure to genre.
