## Component 1 — Misconception Register

### MC-ANY-STROKE-ORDER-WORKS
- **Trigger signal:** Student produces a recognizable letter shape but via an inconsistent, inefficient, or backward stroke sequence (e.g., drawing "O" starting from the bottom and going counter-clockwise, or drawing vertical strokes bottom-to-top), and resists correction because "it looks the same anyway."
- **Conflict evidence [P28]:** "Your 'O' looks fine right now — but let's try writing five 'o's in a row, fast, the way you just did it. Now watch me do it my way, fast. Whose hand gets tired first, and whose letters stay more even and joined-up?"
- **Bridge text [P30]:** "Stroke order and direction don't just matter for how a single letter looks now — they matter for how smoothly, quickly, and legibly you'll write later, especially once letters need to connect (cursive) or you write for a long time. The 'right' order is the one that sets you up for that."
- **Replacement text [P31]:** "Always start each letter at its designated starting point and follow the standard stroke direction, even if an alternate order produces a similar-looking result right now."
- **Discrimination pairs [P33]:**
  - Standard top-to-bottom vertical stroke vs. bottom-to-top (same visual result at slow speed, diverges badly at speed/cursive).
  - Standard clockwise/counter-clockwise convention for circular letters (e.g., "c," "o," "a" all start top-right, move counter-clockwise) vs. an arbitrary starting point — consistency across letters that share a curve shape matters for motor-pattern transfer.
- **s6_path:** "It's reasonable to think 'if it looks right, it IS right' — but handwriting is a motor skill, not just a visual one, and the habits we build now either help or hurt every letter you write for years. We'll build the efficient habit from the start."

### MC-SIZE-AND-BASELINE-DONT-MATTER
- **Trigger signal:** Student writes letters at inconsistent sizes or ignores the baseline/writing lines entirely (letters floating above the line, or wildly varying height between letters in the same word), treating handwriting lines as decorative rather than functional guides.
- **Conflict evidence [P28]:** "Look at this word you wrote — some letters sit on the line, some float above it, and the tall letters aren't all the same height. Now look at this printed word in a book — do the letters all sit on the same line and match up in height?"
- **Bridge text [P30]:** "The baseline and the other guide-lines exist to keep every letter the same size and sitting in the same place — this is what makes handwriting easy for OTHER PEOPLE (and future-you) to read quickly, not just a rule for its own sake."
- **Replacement text [P31]:** "Always place letters so their base sits exactly on the baseline, and check tall/short letter categories (ascenders, x-height, descenders) against the guide lines before moving on."
- **Discrimination pairs [P33]:**
  - A word with all letters correctly baseline-anchored and proportionally sized vs. the same word with letters floating/varying — legibility difference made explicit.
  - "b" (ascender, tall) vs. "a" (x-height, short) vs. "p" (descender, dips below baseline) — three different height categories that must be visually distinguished.
- **s6_path:** "Lines on the page can feel like they're just there to make things harder — but they're actually a shortcut that keeps your writing consistent without you having to judge size by eye every single time."

## Component 2 — Prerequisite Diagnostic Block

**PD-1 [P41] — Letter-recognition readiness**
Prompt: "Show me the letter 'm'. Now show me 'a'."
- Pass: correctly identifies both letters by sight (per `eng.phonics.alphabet-recognition`).
- Fail → [P52]: brief alphabet-recognition refresher before proceeding to formation practice. Persistent failure → route to eng.phonics.alphabet-recognition for reteaching.

---

## Component 3 — Concrete Anchor [P06]

**Anchor scene — Sky-writing and finger-tracing**

> Before any pencil touches paper, have the student "sky-write" a letter in large motions in the air while narrating the stroke path aloud ("down, and around" for a lowercase "a"). Then trace the same letter with a finger on a large printed/textured template (e.g., sandpaper letter or a thick outline), still narrating the stroke path. Only after both steps does the student pick up a pencil. This anchors correct motor sequencing before fine-motor precision is required.

---

## Component 4 — Conceptual Development Sequence

**TA-1 — Stroke-Path Narration [C]**

For a small set of letters (starting with straight-line letters: l, t, i), student narrates the stroke path aloud while sky-writing or tracing ("start at the top, pull straight down").

**TA-2 — Guided Tracing on Lines [C]**

Student traces pre-printed dotted letters on ruled paper (with baseline, midline, and top-line marked), following correct stroke order, then narrates what each line is for (baseline = where letters sit, x-height line = top of short letters).

**TA-3 — Independent Formation with Line Support [C→P]**

Student writes letters independently on ruled paper without a dotted guide, self-checking against baseline/height rules learned in TA-2.

**TA-4 — Letter Category Sorting by Height [P]**

Student sorts a mixed set of letters into three groups by their height category: ascenders (b, d, h, k, l, t — extend above x-height), x-height only (a, c, e, m, n, o, etimes r, s, u, v, w, x, z), and descenders (g, j, p, q, y — dip below baseline), reinforcing the size/baseline rule from Component 1.

**TA-5 — Word-Level Formation with Spacing [P]**

Student writes a short word, applying correct stroke order for each letter AND consistent spacing between letters and between words (using a finger-width or spacer tool as a concrete spacing anchor).

## Component 5 — Worked Examples

**WE-1 (Foundational — stroke narration)**

> "Sky-write the letter 't' and tell me the stroke path out loud."

Target response: student traces a vertical line top-to-bottom in the air, then a horizontal cross-stroke, narrating "down, then across."

**WE-2 (Foundational — height category)**

> "Is the letter 'p' an ascender, x-height, or descender?"

Target response: "descender" — it dips below the baseline.

**WE-3 (Intermediate — word-level formation)**

> "Write the word 'cat' on the lines, making sure every letter sits correctly."

Target response: all three letters correctly baseline-anchored, "c" and "a" at x-height, "t" as a short ascender, with even spacing.

## Component 6 — Mastery Probe Set

**MP-1 [P36] — Stroke order, novel letter**
"Show me how to form the letter 'd' — what's the first stroke?"
*Target:* correct standard stroke order (e.g., circle first, then the tall vertical stroke, per the standard curriculum convention being followed), not just a correct final shape.

**MP-2 [P36] — Baseline/height self-check**
"Look at this word you wrote. Which letters, if any, aren't sitting correctly on the baseline?"
*Target:* student independently identifies any misaligned letters without being told which ones to check.

**MP-3 [P36] — Height category classification**
"Sort these letters by height: g, m, l, p, a, k."
*Target:* correctly groups g/p as descenders, l/k as ascenders, m/a as x-height.

**MP-4 [P36] — Independent formation, novel word**
"Write the word 'jump' on the lines."
*Target:* legible, correctly stroke-ordered, baseline-anchored, with correct ascender (j... actually j is a descender — verify: student correctly forms all 4 letters with j and p as descenders, m at x-height, u at x-height) and consistent spacing.

**MP-5 [P36] — Explain the purpose of guide lines**
"In your own words: why do we write on lines with a baseline and other markings, instead of just writing anywhere on blank paper?"
*Target:* explains that lines keep letter size/placement consistent, which makes writing easier for others (and future-self) to read.

## Component 7 — Session Architecture

```
[P01] Session open — sky-writing warm-up on 2-3 known letters
  ↓
[P41] PD-1 (letter-recognition readiness)
  ↓ PASS
[P06] Anchor: sky-writing + finger-tracing (before pencil touches paper)
  ↓
TA-1: Stroke-path narration [C]
  ↓
[P28] Conflict: inconsistent/backward stroke order → MC-ANY-STROKE-ORDER-WORKS (if triggered)
  ↓
TA-2: Guided tracing on lines [C]
  ↓
[P28] Conflict: baseline/size disregard → MC-SIZE-AND-BASELINE-DONT-MATTER (if triggered)
  ↓
TA-3: Independent formation with line support [C→P]
  ↓
TA-4: Letter category sorting by height [P]
  ↓
TA-5: Word-level formation with spacing [P]
  ↓
[P51] Check-in WE-1 → WE-2 → WE-3
  ↓
[P36] MP-1 through MP-5 (mastery probe set)
  ↓
[P62] Retrieval seed: "Tomorrow, write your name and check that every letter sits on the line correctly."
[P68] Session close
[P85] Regulation tail — praise correct process (stroke order, line-checking) over just final appearance
[P89] Retrieval schedule: 1 day, 3 days, 7 days
[P91] Mastery gate: 80%; re-route to Component 3 anchor if below
```

**Protocol routing:** S0 (no prior pencil-formation habits): full sky-writing/tracing sequence for every new letter, no shortcuts. S1 (has existing but inconsistent habits, e.g. self-taught letter formation): expect more resistance to MC-ANY-STROKE-ORDER-WORKS since existing habits feel "already correct" — use the speed/fatigue conflict evidence explicitly and patiently, allow extra repetition to overwrite the existing motor pattern. S6 (frustrated by fine-motor difficulty or perfectionism about "messy" letters): explicitly separate "correct process" (stroke order, starting point, line-checking) from "neat result" as two different, independently praiseable things — neatness improves with practice, but process correctness is checkable immediately. S9 (L1 uses a different script, e.g. Arabic, Devanagari, Chinese, with different directionality/stroke conventions): treat Latin-script letter formation as a fully new motor skill, explicitly note any conflicting directional habits (e.g. right-to-left writing direction) without treating them as errors — they are a different, valid system being set aside for this specific skill.

---

## Component 8 — Adaptive Flags

- **Process before product**: praise and assess correct stroke order/direction and line-placement explicitly as a separate, checkable criterion from how "neat" the final letter looks — a correct process with imperfect neatness is closer to mastery than a neat letter formed the wrong way.
- **Pre-pencil rehearsal is not optional**: the sky-writing/tracing sequence in Component 3 should precede pencil-and-paper attempts for every genuinely new letter, not just the first one taught.
- **Existing incorrect habits are harder to unlearn than new ones are to build**: for S1-type learners with pre-existing inconsistent formation, budget extra repetition and expect slower initial progress than a true S0 beginner — this is expected, not a sign of a different problem.
- **Script-transfer neutrality**: for S9 learners transferring from a non-Latin script, frame Latin letter-formation conventions as a new, additional skill, never as a correction of the learner's L1 writing system.
